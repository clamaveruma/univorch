\begin{abstract}
% TODO write and trim to 150-300 words.
This Bachelor's Thesis presents a proof-of-concept universal
virtual-machine orchestrator aimed at teaching and research
environments. The system exposes a common abstraction layer that lets
administrators manage virtual machines regardless of the underlying
hypervisor. The design integrates, in a single model, a tree of
resource descriptors with cascade inheritance, an operation engine
based on the Command pattern, and a REST API as the public boundary.
The architecture is split into two layers: a generic, reusable core,
and a domain-specific application for deploying classroom environments
built as a client on top of that core.

The system is delivered as a downloadable Docker container that the
administrator installs through a single-line installer, together with
a command-line client supporting both an interactive REPL and one-shot
invocations. Adaptation to each hypervisor goes through pluggable
connectors; this version ships a mock connector that exercises the
full model without real infrastructure, providing a foundation for the
real VMware and Proxmox connectors planned as the continuation of the
work.

\end{abstract}
